Необходимо обновить функционал раздела «Мої запити» и привести его навигацию к единому UX, используемому в остальных разделах бота.
Требуемое поведение

1. Экран списка запросов
   При переходе пользователя в раздел «Мої запити» необходимо отобразить список всех запросов текущего пользователя.
   Для каждого запроса вывести:
   текущий статус;
   краткое описание.
   Каждый запрос должен отображаться отдельной InlineKeyboard кнопкой.
   Если запросов нет, показать сообщение:
   📭 У вас поки немає створених запитів.

2. Навигация
   При входе в раздел «Мої запити»
   Основная ReplyKeyboard должна измениться и содержать только кнопку:
   🏠 Головне меню
   После выбора конкретного запроса
   После нажатия пользователем на запрос необходимо открыть экран с полной информацией.
   При этом основная ReplyKeyboard должна измениться и содержать только кнопку:
   ⬅️ Назад
   При нажатии «⬅️ Назад» пользователь должен вернуться к списку своих запросов без перехода через главное меню.
   При нажатии «🏠 Головне меню» пользователь должен выйти из раздела и восстановить стандартную навигацию.
   Важно: изменять навигацию только через основную ReplyKeyboard. Не изменять существующие InlineKeyboard.

3. Экран подробной информации
   После выбора запроса необходимо отобразить полную информацию:
   ID запроса;
   статус;
   описание;
   Applicant;
   Executor (если назначен);
   дата создания;
   дата обновления;
   комментарий (feedback), если он существует;
   другая информация, которая уже используется в проекте.
   Экран должен быть только информационным и использовать существующий механизм отображения данных.
   Требования
   Использовать существующую архитектуру проекта.
   Не дублировать бизнес-логику.
   Переиспользовать существующие сервисы получения запросов.
   Не нарушить работу других разделов бота.
   Проверить
   Главное меню → Мої запити.
   Отображение списка запросов.
   Изменение основной ReplyKeyboard на «🏠 Головне меню».
   Переход в карточку запроса.
   Изменение основной ReplyKeyboard на «⬅️ Назад».
   Возврат к списку запросов по кнопке «⬅️ Назад».
   Возврат в главное меню по кнопке «🏠 Головне меню».
   Корректную работу callback, message_id и навигации после многократных переходов.
   Формат результата
   Кратко опиши внесённые изменения.
   Перечисли изменённые файлы.
   Опиши новую навигацию раздела «Мої запити».
   Подтверди, что возврат к списку запросов выполняется без повторной загрузки раздела через главное меню и не нарушает существующую архитектуру проекта.

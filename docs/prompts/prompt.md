Необходимо исправить функционал раздела «🆕 Нові завдання» для пользователей с ролью EXECUTOR.

Проблема
При переходе в раздел «🆕 Нові завдання» одновременно отображаются два списка задач:
Статический текстовый список.
Список задач в виде InlineKeyboard.
Это приводит к дублированию информации и некорректному пользовательскому интерфейсу.

1. Исправить отображение списка задач
   Необходимо полностью убрать статический список.
   Оставить только один список задач, реализованный через InlineKeyboard.
   Для каждой задачи отображать:
   статус;
   краткое описание.
   Каждая задача должна быть отдельной InlineKeyboard кнопкой.
   Если новых задач нет, вывести сообщение:
   📭 Нових завдань немає.
2. Изменить навигацию
   При переходе в раздел «🆕 Нові завдання»
   Основная ReplyKeyboard должна измениться и содержать только кнопку:
   🏠 Головне меню
   После открытия конкретной задачи
   После нажатия пользователем на InlineKeyboard с задачей:
   открыть карточку задачи;
   не назначать задачу автоматически.
   Основная ReplyKeyboard должна измениться и содержать только кнопку:
   ⬅️ Назад
   При нажатии «⬅️ Назад» пользователь должен вернуться к списку новых задач без перехода через главное меню.
   Важно: изменять только основную ReplyKeyboard. Не изменять существующую логику InlineKeyboard.
3. Просмотр задачи
   После выбора задачи необходимо открыть экран с полной информацией.
   Отобразить:
   ID задачи;
   статус;
   описание;
   Applicant;
   дата создания;
   приоритет (если используется);
   другая информация, уже отображаемая в проекте.
   На экране задачи отобразить InlineKeyboard с кнопкой:
   ✅ Прийняти
4. Принятие задачи
   Только после нажатия кнопки:
   ✅ Прийняти
   необходимо:
   назначить текущего пользователя исполнителем (Executor);
   изменить статус задачи согласно бизнес-логике проекта;
   сохранить изменения в базе данных;
   выполнить синхронизацию с Notion (если интеграция уже реализована);
   показать сообщение об успешном принятии задачи;
   вернуть пользователя в раздел «⏳ Мої завдання» или выполнить переход согласно существующей архитектуре.
   До нажатия кнопки «✅ Прийняти» никаких изменений в задаче происходить не должно.
   Проверить
   отображается только один список задач;
   отсутствует статический список;
   при входе в раздел навигация меняется на «🏠 Головне меню»;
   при выборе задачи открывается карточка без назначения исполнителя;
   на карточке отображается кнопка «✅ Прийняти»;
   основная навигация меняется на «⬅️ Назад»;
   кнопка «⬅️ Назад» возвращает к списку новых задач;
   задача назначается исполнителю только после подтверждения;
   после принятия задача исчезает из списка новых задач и появляется в разделе «⏳ Мої завдання»;
   корректно работают ReplyKeyboard, InlineKeyboard, ChatUIService, FSM и message_id.
   Требования
   Использовать существующую архитектуру проекта.
   Не дублировать бизнес-логику.
   Переиспользовать существующие сервисы получения и назначения задач.
   Не нарушить работу других разделов бота.
   Внести минимально необходимые изменения без переписывания архитектуры.
   Формат результата
   Root Cause Analysis проблемы с двойным отображением списка.
   Перечислить изменённые файлы.
   Описать новый сценарий просмотра и принятия задачи.
   Подтвердить, что задача назначается исполнителю только после нажатия «✅ Прийняти», а навигация соответствует новому UX.

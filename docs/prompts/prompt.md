Необходимо исправить баг в разделе «✅ Підтвердити запит».

Проблема

При переходе в раздел «✅ Підтвердити запит» список задач отображается корректно.

Однако при нажатии на любую задачу возникает ошибка:

greenlet_spawn has not been called; can't call await_only() here.
Was IO attempted in an unexpected place?
https://sqlalche.me/e/20/xd2s

Из-за этого карточка задачи не открывается.

Необходимо
Найти точную причину возникновения ошибки.
Определить, в каком месте происходит обращение к БД вне корректного async-контекста.
Исправить проблему без изменения существующей бизнес-логики.
Проверить

Особое внимание уделить следующим моментам:

используется ли Lazy Loading (lazy="select") при обращении к связанным сущностям;
происходит ли обращение к relationship после закрытия AsyncSession;
не вызывается ли ORM-свойство во время рендера сообщения;
не передается ли ORM-модель между слоями после завершения сессии;
используются ли joinedload, selectinload или другие eager-loading стратегии там, где они необходимы;
не выполняется ли дополнительный SQL-запрос внутри свойства модели или метода форматирования текста.
Требования
Не использовать обходные решения.
Не скрывать исключение через try/except.
Не отключать lazy loading глобально без необходимости.
Сначала определить Root Cause, затем внести минимально необходимые изменения.
Если проблема связана с загрузкой связанных сущностей, использовать корректную стратегию eager loading (selectinload / joinedload) только там, где это действительно требуется.
Не нарушить существующую архитектуру проекта.
Формат результата
Root Cause Analysis — почему возникает greenlet_spawn has not been called.
Какие ORM-сущности или relationship вызывают дополнительный запрос.
Какие файлы были изменены.
Какие изменения внесены.
Почему предложенное решение полностью устраняет ошибку и не влияет на остальные сценарии работы бота.

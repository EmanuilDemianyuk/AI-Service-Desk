Необходимо реализовать полноценную интеграцию задач с Notion.
Основной принцип
Единым источником правды (Single Source of Truth) должна оставаться база данных приложения.
Notion используется только как внешняя система отображения и синхронизации.
Все изменения должны сначала успешно фиксироваться в базе данных приложения, и только после успешного commit выполняться синхронизация с Notion.
При ошибке синхронизации с Notion изменения в базе данных не должны откатываться.

1. Создание задачи
   При создании новой задачи в Telegram-боте необходимо:
   Создать задачу в базе данных.
   Успешно выполнить commit.
   Только после успешного сохранения создать соответствующую страницу в Notion.
   Сохранить notion_page_id в базе данных.
   Если создание страницы в Notion завершилось ошибкой:
   не откатывать транзакцию БД;
   записать ошибку в лог;
   оставить возможность повторной синхронизации.
2. Обновление задачи
   Любое изменение задачи в Telegram-боте должно сначала:
   обновить запись в базе данных;
   выполнить commit;
   затем обновить соответствующую страницу в Notion.
   Необходимо синхронизировать любые изменения, включая:
   описание;
   статус;
   Applicant;
   Executor;
   тип Executor;
   feedback;
   дату завершения;
   подтверждение Applicant;
   любые другие поля, отображаемые в Notion.
3. Удаление задачи
   При удалении задачи:
   удалить (или пометить удалённой) запись согласно существующей логике проекта;
   выполнить commit;
   затем архивировать страницу в Notion (archived = true).
   Физически удалять страницу из Notion не требуется.
4. Сервис синхронизации
   Создать отдельный сервис, например:
   NotionSyncService
   Сервис должен отвечать исключительно за взаимодействие с Notion.
   Он должен содержать методы:
   create_task()
   update_task()
   archive_task()
   sync_task()
   sync_all_tasks()
   Запрещено вызывать Notion API напрямую из:
   Telegram handlers;
   Service Layer бизнес-логики;
   Repository.
   Взаимодействие с Notion должно быть централизовано через NotionSyncService.
5. Модель данных
   Проверить наличие поля:
   notion_page_id
   Если поле отсутствует:
   создать миграцию;
   обновить ORM-модель;
   обновить DTO;
   обновить Repository.
6. Первичная синхронизация
   После завершения интеграции необходимо выполнить полную синхронизацию существующих данных.
   Алгоритм:
   Получить все задачи из базы данных.
   Получить все страницы из базы данных Notion.
   Выполнить сопоставление записей.
   Если задача отсутствует в Notion:
   создать её.
   Если задача существует:
   обновить все поля в соответствии с базой данных приложения.
   Если страница существует в Notion, но задача отсутствует в базе данных:
   не удалять автоматически;
   вывести такие записи в лог как несинхронизированные (или обработать согласно существующей бизнес-логике проекта).
   В результате база данных приложения должна полностью синхронизировать состояние Notion.
7. Обработка ошибок
   Если Notion недоступен:
   Telegram-бот продолжает работать;
   база данных остаётся актуальной;
   ошибка записывается в лог;
   синхронизация не должна приводить к падению пользовательского сценария.
   Не использовать rollback транзакции БД при ошибках Notion API.
8. Проверить
   Создание
   создать новую задачу;
   убедиться, что запись появилась в БД;
   после этого появилась в Notion;
   notion_page_id сохранён.
   Обновление
   Проверить синхронизацию после изменения:
   статуса;
   исполнителя;
   Applicant;
   feedback;
   подтверждения выполнения;
   возврата в IN_PROGRESS;
   завершения задачи.
   Удаление
   удалить задачу;
   проверить архивирование страницы в Notion.
   Массовая синхронизация
   После запуска sync_all_tasks():
   все задачи из базы данных присутствуют в Notion;
   данные совпадают;
   отсутствуют дубликаты;
   все notion_page_id заполнены корректно.
   Требования
   Использовать существующую архитектуру проекта.
   Не дублировать бизнес-логику.
   Не выполнять запросы к Notion из Telegram handlers.
   Все операции с Notion выполнять только после успешного сохранения изменений в базе данных.
   Не нарушить существующий функционал Telegram-бота.
   Реализовать переиспользуемую архитектуру, позволяющую в будущем добавлять другие внешние интеграции.
   Формат результата
   Кратко опиши архитектуру интеграции.
   Перечисли изменённые файлы.
   Опиши реализацию NotionSyncService.
   Опиши механизм синхронизации после успешного commit в базе данных.
   Опиши реализацию массовой синхронизации существующих задач.
   Подтверди, что база данных приложения остаётся единственным источником правды, а Notion является синхронизируемым внешним представлением данных.

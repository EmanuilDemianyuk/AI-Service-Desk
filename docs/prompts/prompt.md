Обнаружена проблема в работе Telegram-бота.

Сценарий воспроизведения
Пользователь нажимает KeyboardButton(text="👥 Користувачі").
Бот успешно обрабатывает команду и отображает список пользователей.
После этого пользователь нажимает любую другую KeyboardButton.
Ничего не происходит — бот больше не реагирует на нажатия.
Необходимо
Найти причину, почему после выполнения команды 👥 Користувачі обработка последующих KeyboardButton прекращается.
Проверить:
остается ли активной текущая BotCommand;
очищается ли состояние (FSM, State, Session, Context);
не блокируется ли роутер или обработчик сообщений;
корректно ли завершается текущий сценарий после отправки списка пользователей;
не перехватывает ли сообщения middleware, filter или handler с высоким приоритетом.
Требования
После завершения обработки команды 👥 Користувачі пользователь должен иметь возможность сразу выбрать любую другую команду из клавиатуры.
Если команда не является многошаговым сценарием, она должна полностью завершаться и освобождать состояние пользователя.
Если проблема связана с архитектурой обработки BotCommand, предложить и реализовать универсальное решение, которое предотвратит подобные ошибки для всех команд.

Перед внесением изменений проанализируй цепочку обработки входящих сообщений, жизненный цикл BotCommand и управление состоянием пользователя. Определи корневую причину проблемы, а затем внеси минимально необходимые изменения без нарушения существующей архитектуры.

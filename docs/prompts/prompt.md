Необходимо улучшить навигацию в разделе «Створити запит».

Требуемое поведение

При переходе пользователя в раздел «Створити запит» необходимо изменить основную ReplyKeyboard.

Вместо стандартной навигации должна отображаться только одна кнопка:

⬅️ Назад

Важно: изменять навигацию только через основную ReplyKeyboard. Не изменять InlineKeyboard.

Логика кнопки «⬅️ Назад»

При нажатии пользователем кнопки «⬅️ Назад»:

немедленно завершить сценарий создания заявки;
очистить текущее состояние (FSM, State, Session, Context);
удалить все временные данные, накопленные в процессе создания заявки;
восстановить стандартную навигацию пользователя (ReplyKeyboard);
вернуть пользователя в главное меню.
Важно

Кнопка «⬅️ Назад» не должна восприниматься как текст заявки.

Необходимо исключить ситуацию, при которой сообщение:

⬅️ Назад

обрабатывается как описание новой заявки или любое другое пользовательское поле формы.

Обработчик кнопки «⬅️ Назад» должен иметь более высокий приоритет, чем обработчики ввода данных формы.

Проверить
Переход:
Главное меню → Створити запит.
Замену основной навигации на кнопку «⬅️ Назад».
Нажатие «⬅️ Назад» на любом этапе создания заявки.
Полную очистку состояния (FSM).
Восстановление стандартной навигации пользователя.
Отсутствие создания новой заявки после отмены.
Что текст «⬅️ Назад» никогда не сохраняется как содержимое заявки.
Требования
Использовать существующую архитектуру проекта.
Не дублировать обработчики.
Переиспользовать существующий механизм управления навигацией.
Реализовать универсальный механизм отмены, который в будущем можно использовать и в других многошаговых сценариях.
Формат результата
Кратко опиши внесённые изменения.
Перечисли изменённые файлы.
Объясни, как реализована отмена сценария создания заявки.
Подтверди, что кнопка «⬅️ Назад» больше не может быть обработана как текст заявки и всегда корректно завершает текущий сценарий.

Необходимо исправить баг в админ-панели при изменении роли пользователя.

Проблема
В разделе ADMIN → «Змінити роль», при выборе роли:
Виконавець (EXECUTOR)
бот НЕ запрашивает обязательный выбор типа исполнителя (SYSADMIN / MASTER), что приводит к созданию/обновлению пользователя без корректного type.
Ожидаемое поведение
Если ADMIN изменяет роль пользователя на EXECUTOR, система должна:

1. НЕ завершать изменение роли сразу
   После выбора EXECUTOR необходимо перейти на следующий шаг:
   Выбор типа исполнителя
2. Показать InlineKeyboard выбора типа
   Отобразить две кнопки:
   MASTER
   SYSADMIN
3. После выбора типа
   сохранить роль EXECUTOR;
   сохранить выбранный type;
   обновить пользователя в базе данных;
   завершить сценарий изменения роли;
   вернуть пользователя в админ-меню или экран пользователя;
   показать сообщение об успешном обновлении.
   Поведение для других ролей
   Если ADMIN выбирает:
   APPLICANT
   ADMIN
   то:
   шаг выбора type НЕ нужен;
   обновление роли происходит сразу;
   поле type очищается или устанавливается в NULL (в зависимости от модели).
   Корневая проблема (предположительно)
   Необходимо проверить:
   отсутствует ли переход в state выбора executor_type;
   не вызывается ли update пользователя сразу после выбора EXECUTOR;
   не переиспользуется ли логика создания пользователя вместо логики обновления;
   не пропускается ли FSM шаг между выбором роли и выбором type;
   корректно ли обрабатывается callback для EXECUTOR в админ-редактировании.
   Требования к реализации
   Не ломать существующий flow изменения ролей.
   Использовать текущий FSM / state machine админки.
   Переиспользовать существующую логику выбора типа из создания пользователя (если есть).
   Добавить недостающий шаг, а не переписывать весь сценарий.
   Обеспечить обязательность выбора type только для EXECUTOR.
   Проверить сценарии
   EXECUTOR
   ADMIN выбирает «Змінити роль»
   Выбирает EXECUTOR
   Появляется выбор:
   MASTER
   SYSADMIN
   После выбора:
   роль обновляется
   type сохраняется
   сценарий завершается корректно
   APPLICANT / ADMIN
   ADMIN выбирает роль
   Сразу происходит обновление
   Без запроса type
   Формат результата
   Root cause бага.
   Какие места в коде нарушают flow.
   Перечень изменённых файлов.
   Описание исправленного сценария.

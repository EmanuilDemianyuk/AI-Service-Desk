Необходимо исправить баг в функционале «✅ Позначити як завершене».

Проблема

При нажатии пользователем на InlineKeyboard кнопку «✅ Підтвердити завершення» отображается ошибка:

Task <id> is not waiting for applicant confirmation

Однако в данном сценарии подтверждение выполняет исполнитель задачи, а не заявитель. Поэтому эта проверка не соответствует текущему бизнес-процессу.

Необходимо

Найти место, где возникает исключение:

Task <id> is not waiting for applicant confirmation
Проанализировать, почему используется логика подтверждения заявителем вместо логики завершения задачи исполнителем.
Проверить:
какой сервис вызывается при обработке callback;
не используется ли обработчик подтверждения назначения (applicant confirmation) вместо завершения работы;
соответствует ли текущий статус задачи ожидаемому бизнес-процессу;
корректно ли выполняется переход из IN_PROGRESS в сценарий завершения задачи.
Ожидаемое поведение

При нажатии «✅ Підтвердити завершення»:

Не должно выполняться подтверждение заявителем.

Не должна выбрасываться ошибка:

Task <id> is not waiting for applicant confirmation
Пользователь должен перейти к следующему шагу — вводу feedback о выполненной работе.
После успешного ввода feedback задача должна быть переведена в статус COMPLETED.
Требования
Не использовать существующий сценарий applicant confirmation, если он предназначен для другого бизнес-процесса.
При необходимости реализовать отдельный сервис/метод для завершения задачи исполнителем.
Не изменять существующую бизнес-логику подтверждения заявителем, если она используется в других сценариях.
Сначала определить корневую причину ошибки, затем внести минимально необходимые изменения.
Формат результата
Root Cause Analysis.
Где использовался неверный сценарий подтверждения.
Какие файлы были изменены.
Почему предложенное решение корректно разделяет процессы подтверждения назначения и завершения задачи исполнителем

Необходимо улучшить навигацию в разделе «Мої завдання», чтобы она соответствовала принципу иерархической навигации.

Требуемое поведение

1. Экран списка задач

При переходе в раздел «Мої завдання» пользователь должен увидеть:

список своих задач;

На этом экране вместо стандартной навигации должна отображаться только одна кнопка:

🏠 Головне меню

При нажатии на неё пользователь возвращается в главное меню.

2. Экран просмотра задачи

После нажатия пользователем на конкретную задачу (обработчик view_my_task_callback) необходимо открыть экран с полной информацией о задаче.

На этом экране навигация должна измениться.

Вместо кнопки «🏠 Головне меню» необходимо отображать кнопку:

⬅️ Назад

При нажатии кнопки «⬅️ Назад» пользователь должен вернуться к списку своих задач, а не в главное меню.

3. Навигация

Итоговая структура должна быть следующей:

Главное меню
│
▼
Мої завдання
│
├── Task 1
├── Task 2
├── Task 3
│
└── 🏠 Головне меню
│
▼
Главное меню

При выборе задачи:

Главное меню
│
▼
Мої завдання
│
▼
Деталі задачі
│
└── ⬅️ Назад
│
▼
Список задач

Требования к реализации
Не создавать новый экран списка — использовать существующий обработчик раздела «Мої завдання».
Кнопка «⬅️ Назад» должна повторно открывать список задач без повторного перехода через главное меню.

Не создавать дублирующую бизнес-логику.
Реализовать переиспользуемую навигацию, которую в будущем можно применять и в других разделах бота.
После возврата к списку задач навигация должна снова отображать кнопку «🏠 Головне меню».
Что необходимо проверить
Корректную работу callback'ов при переходе:
Главное меню → Мої завдання;
Мої завдання → Деталі задачі;
Деталі задачі → Назад → Мої завдання;
Мої завдання → Головне меню.
Что message_id, callback и состояние пользователя не теряются при переходах.
Что навигация продолжает работать после многократных переходов между списком задач и карточкой задачи.
